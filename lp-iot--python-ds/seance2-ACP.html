ACP¶

Dans le sous-module decomposition, nous allons importer la fonction PCA(). Le fonctionnement de celle-ci est similaire toutes les autres dans scikit-learn.

  1. On cr d'abord un objet (nomm ici pca) qui va contenir le rsultat de l'ACP. Dans la fonction PCA(), on pourra aussi indiquer les paramtres tels que le nombre de composants retenir (n_components - ici on garde tout).
  2. Ensuite, on ajuste (fit en anglais) sur des donnes avec la fonction fit() de l'objet prcdemment cr. Dans cette fonction, nous devons donc passer les donnes utiliser (variables uniquement quantitatives).

Si l'on souhaite une ACP norme (ce qui n'est pas le cas ici), il nous faut standardiser les donnes en amont, avec par exemple la fonction scale() du sous-module preprocessing (importe aussi ici). Bien videmment, il ne faut passer que des variables quantitatives, sans donnes manquantes.

Variance expliquée¶

L'objet pca comprend maintenant un certain nombre d'objets et de fonctions. Le premier objet est le tableau des variances expliques (explained_variance_ - proportionnelles aux valeurs propres) par chaque dimension, et le ratio (proportion) de variance explique par dimension (explained_variance_ratio_).

In [8]:
print(pca.explained_variance_)
print(pca.explained_variance_ratio_)

for j in range(WGI_num.shape[1]):
    axes.arrow(0, 0, pca.components_[0, j], pca.components_[1, j], width = .001, color = "darkblue", alpha = .25)
    if (pca.components_[1, j] > 0):
        va = "bottom"
    else:
        va = "top"
