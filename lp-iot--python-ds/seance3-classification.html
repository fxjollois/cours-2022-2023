Classification¶

Nous allons ici raliser une classification non supervise avec 3 mthodes (CAH, $k$-means et DBSCAN). Bien videmment, ici, nous pouvons peut-tre suppos qu'une partition en 6 classes (1 par continent) est possible. Mais nous n'avons pas de relle ide de ce nombre de classes. Pour l'estimer, nous allons utiliser 2 techniques.

Classification Ascendante Hiérarchique (CAH)¶

La premire possibilit est d'utiliser la CAH, afin d'obtenir un dendrogramme reprsentant les regroupements 2 2 de toutes les classes.

Tout d'abord, nous importons la mthode, puis nous l'appliquons. Ici, on indique qu'on ne souhaite pas raliser de dcoupage au pralable, et donc qu'on souhaite garder toute la hirarchie. Voici pourquoi on choisit 0 pour le paramtre distance_threshold et None pour n_clusters. Le paramtre affinity permet de choisir la distance utilise (euclidean par dfaut - ce qui nous va bien). Le paramtre linkage permet lui de choisir le critre d'aggrgation entre deux classes (ward par dfaut - ce qui nous va bien aussi).

from scipy.cluster.hierarchy import dendrogram

def plot_dendrogram(model, **kwargs):
    # Create linkage matrix and then plot the dendrogram

    # create the counts of samples under each node
    counts = numpy.zeros(model.children_.shape[0])
    n_samples = len(model.labels_)
    for i, merge in enumerate(model.children_):
        current_count = 0
        for child_idx in merge:
            if child_idx < n_samples:
                current_count += 1  # leaf node
            else:
                current_count += counts[child_idx - n_samples]
        counts[i] = current_count

    linkage_matrix = numpy.column_stack([model.children_, model.distances_, counts]).astype(float)

    # Plot the corresponding dendrogram
    dendrogram(linkage_matrix, **kwargs)

Nous utilisons donc cette mthode sur nos pays. On remarque un saut important entre 2 et 3 classes, puis entre 3 et 4 classes (et un peu moindre entre 4 et 5 classes). On imagine donc qu'on pourrait couper la hirarchie ces 2 niveaux (cf graphique). Ainsi, cela suggre qu'on pourrait avoir une partition soit en 2 classes, soit en 3 classes.

Choix du nombre de classes avec $k$-means¶

L'algorithme $k$-means nous permet d'avoir la fin l'inertie intra-classes, qui reprsente la disparit des individus l'intrieur des classes. Plus cette valeur est proche de 0, meilleur est la partition. Malheureusement, la meilleure partition selon ce critre est donc celle avec autant de classes que d'individus (ce qui n'est pas trs utile...).

On va donc chercher un point d'inflexion dans la courbe d'volution de ce critre. Voici comment faire pour avoir ce graphique. Et ici, le point le plus marquant est celui 2 classes. Ensuite, celui 3 classes peut montrer aussi une certaine cassure dans l'volution du critre.

In [25]:
plt.figure(figsize = (16, 6))
inertia = []
for k in range(1, 11):
    kmeans = KMeans(n_clusters = k, init = "random", n_init = 20).fit(WGI_num)
    inertia = inertia + [kmeans.inertia_]
inertia = pandas.DataFrame({"k": range(1, 11), "inertia": inertia})
seaborn.lineplot(data = inertia, x = "k", y = "inertia")
plt.scatter(2, inertia.query('k == 2')["inertia"], c = "red")
plt.scatter(3, inertia.query('k == 3')["inertia"], c = "red")
plt.show()
